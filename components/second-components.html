Hanz and Franz

Hanz and Franz began as a practice in manipulating the DOM, or specifically a progress bar. Once I had the simple JavaScript code written for a one player game, I decided to take it to the next level by including an additional player.
Each player has two attack modes and two defenses. Later I decided the second player shouldn't have to control the defensive moves. So I wrote a random function that has two defenses and two non-reactive responses. Depending on the attack and defense, the health of the players varies.
I began with simple alerts. Unsatisfied with this solution, I designed a vintage sideshow with two humorous fighters.
At first I researched sprites. Once I had several put together and included in my game, I realized I didn't like the way they worked or looked. So I did more research and learned to create giffs. Then I simply had to figure out how to get a gif to work "once" on command. That turned out to be pretty difficult. I finally figured out how to replace the gif with a png each time an attack was triggered. Then I had to set a delay so that defense didn't happen at the exact same time. The delays also caused a problem. Mainly if you push the button too fast.
Honestly the whole game is campy and glitchy, so I decided it was ok for my first game.
Also, at this point I had already designed the game for a medium to large screen. As I began carefully placing each of the 10 giffs and pngs, I realized the game was not going to be responsive for other devices. That was unfortunate, but it was also one of my first projects, so I decided to leave it.
Well, I hope you enjoy Hanz and Franz. My favorite giff is when Franz is knocked-out.

To-Dos and Calculators

This began as a simple javascript calculator.
I wanted a drop-down menu where you could choose the type of conversion, instead of input fields and buttons. Once I had accomplished that, we were already working in UI-Router and creating to-do lists.
It seemed like it would be good practice to create several routes, controllers, and to-do lists. Once I had a theme, I decided to include the calculator as part of the app.
I spent many hours making the images pixal perfect. I included scroll-bars that started and stopped on exact points! (Later I would learn to use 'position absolute' with percentages. So, this is not a mobile friendly design.) But I am very proud of it! I believe it is beautiful!

CSS Resume

It might be obvious how much love went into this app.
I found an amazing creative team I wanted to apply too, so I decided to create a resume that would showcase my creativity and versatility. I designed every image to be beautuful, a little whimsical, yet slightly dark. I didn't want it to be too cute, but I wanted to create a stage ready to employ characters later.

This project has paralax, Angular components, themed pages, gifs, CSS animations and transition, some jQuery, bootstrap, and much more! I really wanted to dive into CSS. This app is scalable and responsive! I watched many tutorials and read a lot of blogs in order to come away with this variety of amazing animations. Although, I felt like maybe this site highlighted CSS a little better than it packaged myself and my skill set. So I decided to start a new resume site. But I love what is going on here, and plan to create a character that can be showcased in this environment.
I hope you enjoy it as much as I do.

Pop-Up Devil

I wanted to take the "whack-a-mole" game and switch it up. I made a series of wind-up "Jack-in-the-box" using Photoshop. There are three colors- red, blue, and green. Each color has the possibility of launching a monkey or a devil.
As the boxes play their music and transition between four winding images, the player can click on any box. The question is- will they find the monkey or the devil?
This game is still under construction.

The Zoo Keeper

The Zoo Keeper is a game I designed that deals with space, objects, and organizing. In this specific game, their are seven environment, which the zoo keeper can drag and drop animals into. I plan to have a series of if statements that will allow you to leave an animal anywhere you like. If you choose wisely you will get points. If you choose unwisely you will get warnings, and eventually there will be a consequence.
This project is still under construction.
I see this project as having real life application. It could be changed into a storage app.
